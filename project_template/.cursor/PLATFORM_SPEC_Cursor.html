Covers .cursor/ folder layout, all supported primitives (rules/skills/agents/commands/hooks/MCP), parity tiers, vault doc location, crawl freshness gate, and install verification. setting up Cursor in a gald3r project, authoring rules/skills/agents/commands/hooks, understanding .cursor/ structure, verifying Cursor parity, or answering questions about Cursor's capabilities.

What Makes This Platform Different

  • Rules use the Cursor-only .mdc extension (all other platforms use .md); parity sync maps the extension automatically.
  • hooks.json lives at .cursor/hooks.json (repo .cursor/ root), not inside .cursor/hooks/; _hook_md fields point at companion .md files inside .cursor/hooks/.
  • No equivalent of AGENTS.md enforcement engine — always-apply behavior comes from .cursor/rules/*.mdc (alwaysApply: true).
  • Hook execution: PowerShell on Windows; cross-platform invocation must be honored for Unix users.
  • MCP is configured via Cursor settings / root .mcp.json (gitignored) — there is no committed .cursor/mcp.json in this repo.

Hook System

  • Type: json_config
  • Config file: .cursor/hooks.json (repo-root .cursor/, version 1 — NOT inside .cursor/hooks/)
  • Events available (21 total, verified 2026-05-31 against https://docs.cursor.com/agent/hooks):

Agent hooks (fire during Cmd+K / Agent Chat sessions):

  • sessionStart / sessionEnd — session lifecycle
  • preToolUse / postToolUse / postToolUseFailure — generic tool use (all tools)
  • subagentStart / subagentStop — Task tool subagent lifecycle
  • beforeShellExecution / afterShellExecution — shell command control
  • beforeMCPExecution / afterMCPExecution — MCP tool control
  • beforeReadFile / afterFileEdit — file access and edit control
  • beforeSubmitPrompt — validate prompts before submission
  • preCompact — observe context window compaction
  • stop — agent completion
  • afterAgentResponse / afterAgentThought — track responses

Tab hooks (fire for autonomous inline completions):

  • beforeTabFileRead — control file access for Tab completions
  • afterTabFileEdit — post-process Tab edits

App lifecycle hooks (fire outside any agent session):

  • workspaceOpen — fires on workspace open and every folder change; can return additional plugin paths
  • Event payload format: JSON (hooks return the { continue = true } envelope; block via exit code / verdict)
  • Limitations: command is a full PowerShell invocation; optional matcher regex over tool names; _hook_md companion-doc path (T1171). No async; HTTP hook type (T600) lives in .claude/hooks/ pending parity.
  • gald3r hook files: g-hk-session-start.py (sessionStart); g-hk-agent-complete.py, g-hk-nightly-learn.ps1, g-hk-session-end.py (stop); g-hk-validate-shell.py (beforeShellExecution); g-hk-pre-tool-call-gald3r-guard.py, g-hk-pre-tool-call-prd-freeze.py, g-hk-pre-tool-call-member-gald3r-guard.py (preToolUse)

Hook API Reference

Full input/output schemas for each hook event, verified 2026-05-31 from https://docs.cursor.com/agent/hooks.

Hooks configuration format

{
  "version": 1,
  "hooks": {
    "sessionStart":         [{ "command": "./hooks/script.ps1" }],
    "sessionEnd":           [{ "command": "./hooks/script.ps1" }],
    "preToolUse":           [{ "command": "./hooks/script.ps1", "matcher": "Shell|Read|Write" }],
    "postToolUse":          [{ "command": "./hooks/script.ps1" }],
    "postToolUseFailure":   [{ "command": "./hooks/script.ps1" }],
    "subagentStart":        [{ "command": "./hooks/script.ps1" }],
    "subagentStop":         [{ "command": "./hooks/script.ps1" }],
    "beforeShellExecution": [{ "command": "./hooks/script.ps1", "failClosed": true }],
    "afterShellExecution":  [{ "command": "./hooks/script.ps1" }],
    "beforeMCPExecution":   [{ "command": "./hooks/script.ps1", "failClosed": true }],
    "afterMCPExecution":    [{ "command": "./hooks/script.ps1" }],
    "beforeReadFile":       [{ "command": "./hooks/script.ps1" }],
    "afterFileEdit":        [{ "command": "./hooks/script.ps1" }],
    "beforeSubmitPrompt":   [{ "command": "./hooks/script.ps1" }],
    "preCompact":           [{ "command": "./hooks/script.ps1" }],
    "stop":                 [{ "command": "./hooks/script.ps1", "loop_limit": 10 }],
    "afterAgentResponse":   [{ "command": "./hooks/script.ps1" }],
    "afterAgentThought":    [{ "command": "./hooks/script.ps1" }],
    "beforeTabFileRead":    [{ "command": "./hooks/script.ps1" }],
    "afterTabFileEdit":     [{ "command": "./hooks/script.ps1" }],
    "workspaceOpen":        [{ "command": "./hooks/script.ps1" }]
  }
}

Optional per-entry fields: matcher (regex over tool name), failClosed (bool, default fail-open), loop_limit (int, for stop), _hook_md (companion doc path, T1171).

Agent Hooks — payload schemas

sessionStart — fire-and-forget; no input/output fields defined.

sessionEnd — fire-and-forget:

// input
{ "session_id": "...", "reason": "completed|aborted|error|window_close|user_close",
  "duration_ms": 12345, "is_background_agent": false, "final_status": "...",
  "error_message": "(optional, when reason=error)" }
// output: none (fire-and-forget)

preToolUse — can allow/deny/modify tool calls:

// input
{ "tool_name": "Shell", "tool_input": { "command": "npm install", "working_directory": "/project" },
  "tool_use_id": "abc123", "cwd": "/project", "model": "claude-sonnet-4-...", "agent_message": "..." }
// output
{ "permission": "allow|deny", "user_message": "(shown when denied)",
  "agent_message": "(sent to agent when denied)", "updated_input": { "command": "npm ci" } }

postToolUse — fire-and-forget after tool completes.

postToolUseFailure — fire-and-forget on tool failure:

// input
{ "tool_name": "Shell", "tool_input": { "command": "npm test" }, "tool_use_id": "abc123",
  "cwd": "/project", "error_message": "Command timed out after 30s",
  "failure_type": "timeout|error|permission_denied", "duration": 5000, "is_interrupt": false }
// output: none

subagentStart / subagentStop — fire-and-forget; no defined payload.

beforeShellExecution — can allow/deny shell commands:

// input
{ "command": "ls -l", "cwd": "/app", "sandbox": true }
// output
{ "permission": "allow|deny|ask", "user_message": "...", "agent_message": "..." }

Set failClosed: true for security-critical hooks (default: fail-open).

afterShellExecution — fire-and-forget after shell runs.

beforeMCPExecution — can allow/deny MCP tool calls:

// input (stdio server)
{ "tool_name": "readFile", "tool_input": "{\"path\": \"/etc/passwd\"}", "command": "mcp-server" }
// input (HTTP server)
{ "tool_name": "readFile", "tool_input": "{\"path\": \"/etc/passwd\"}", "url": "http://localhost:3000" }
// output
{ "permission": "allow|deny|ask", "user_message": "...", "agent_message": "..." }

afterMCPExecution — fire-and-forget.

beforeReadFile — can allow/deny file reads sent to model:

// input
{ "file_path": "/abs/path/file.py", "content": "<full file contents>",
  "attachments": [{ "type": "file|rule", "file_path": "/abs/path" }] }
// output
{ "permission": "allow|deny", "user_message": "(shown when denied)" }

Failures are logged and read is allowed through by default; use failClosed: true to block.

afterFileEdit — fire-and-forget after agent edits a file.

beforeSubmitPrompt — can block prompt submissions:

// input
{ "prompt": "<user prompt text>",
  "attachments": [{ "type": "file|rule", "file_path": "/abs/path" }] }
// output
{ "continue": true|false, "user_message": "(shown when blocked)" }

preCompact — observe context window compaction; fire-and-forget.

stop — fires when agent completes; fire-and-forget. Use loop_limit to cap recursive stop-hook invocations.

afterAgentResponse — fire-and-forget:

// input
{ "text": "<assistant final text>" }
// output: none

afterAgentThought — fire-and-forget:

// input
{ "text": "<aggregated thinking text>", "duration_ms": 5000 }
// output: none

Tab Hooks — payload schemas

beforeTabFileRead — can allow/deny file reads for Tab completions:

// input (no attachments field — Tab doesn't use prompt attachments)
{ "file_path": "/abs/path/file.py", "content": "<full file contents>" }
// output
{ "permission": "allow|deny" }

afterTabFileEdit — fire-and-forget after Tab edits a file.

App Lifecycle Hooks

workspaceOpen — fires when workspace opens and on every folder change:

// output (optional)
{ "plugin_paths": ["./additional-plugin-path"] }

Can return additional plugin paths to load for the current workspace.
